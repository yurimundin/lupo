# Lupo v0.1.1

Release incremental para validação controlada no Windows.

## Destaques

- Lixeira com ciclo completo: mover itens, restaurar e esvaziar definitivamente
  com rollback em memória se a gravação falhar.
- Ícones modernos para pastas usando Lucide: clique com o botão direito em um
  grupo e use "Alterar ícone". A escolha fica salva como metadado próprio do
  Lupo, sem alterar o `IconID` padrão do KeePass.
- Recuperação guiada ao detectar `.kdbx.tmp` pendente depois de crash ou
  gravação interrompida.
- Restauração conservadora de `.kdbx.bak`: antes de substituir o cofre atual,
  o app preserva uma cópia `before-restore`.
- Abertura normal não é interrompida por `.bak` sozinho, porque esse arquivo
  é esperado após salvamentos bem-sucedidos.
- Versão sincronizada para novos instaladores Windows `.msi` e `.exe`.

## Segurança e compatibilidade

- Salvamento atômico preserva o cofre atual e mantém `.kdbx.bak` como a versão
  anterior ao último save.
- Recuperação de backup só acontece por ação explícita do usuário.
- Ícones de pastas continuam compatíveis com KeePass/KeePassXC: outros clientes
  ignoram o metadado visual do Lupo e abrem o mesmo `.kdbx` normalmente.

## Notas importantes

- Sem assinatura de código nesta versão. O Windows SmartScreen pode exibir
  aviso ao abrir o instalador.
- Sem auto-update. Atualizações futuras devem ser baixadas manualmente.
- Mantenha backup externo dos cofres usados em teste.

## Smoke flow recomendado

1. Instalar em uma máquina Windows limpa ou perfil limpo.
2. Abrir o app.
3. Criar ou abrir um cofre `.kdbx`.
4. Criar uma entrada, salvar, fechar e reabrir.
5. Mover uma entrada para a Lixeira, restaurar e depois testar esvaziar a
   Lixeira.
6. Alterar o ícone de uma pasta pelo menu de contexto e reabrir o cofre para
   confirmar persistência.
7. Criar um arquivo `.kdbx.tmp` ao lado do cofre e confirmar que o aviso de
   recuperação aparece.
8. Confirmar que "Abrir cofre atual" segue sem restaurar backup.
9. Confirmar que "Restaurar backup" preserva o cofre atual em
   `.before-restore-*.bak`.
